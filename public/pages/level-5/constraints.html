SQL Constraints – Enforce Data Integrity

Constraints protect your data by enforcing rules on tables and columns.

What are SQL Constraints?

SQL constraints are rules applied to table columns to ensure valid, reliable, and consistent data. They automatically prevent incorrect or conflicting entries at the database level.

  • Ensure primary keys are unique
  • Prevent NULLs in required columns
  • Restrict values via conditions (e.g. age > 0)
  • Maintain relationships between tables

Common SQL Constraints

CREATE TABLE TableName (
  Column1 datatype CONSTRAINT constraint_name TYPE,
  ...
);

Example 1 – PRIMARY KEY on Customers

CREATE TABLE Customers (
  CustomerID CHAR(5) PRIMARY KEY,
  CompanyName VARCHAR(40) NOT NULL
);

Example 2 – FOREIGN KEY in Orders

CREATE TABLE Orders (
  OrderID INT PRIMARY KEY,
  CustomerID CHAR(5),
  FOREIGN KEY (CustomerID) REFERENCES Customers(CustomerID)
);

Example 3 – NOT NULL on Product Name

CREATE TABLE Products (
  ProductID INT,
  ProductName VARCHAR(50) NOT NULL
);

Example 4 – CHECK Constraint on UnitPrice

CREATE TABLE Products (
  ProductID INT,
  UnitPrice DECIMAL(10,2),
  CHECK (UnitPrice >= 0)
);

Example 5 – UNIQUE Constraint on Email

CREATE TABLE Employees (
  EmployeeID INT,
  Email VARCHAR(100) UNIQUE
);

Key SQL Constraints Summary

  • PRIMARY KEY – uniquely identifies each record
  • FOREIGN KEY – maintains referential integrity
  • NOT NULL – ensures a value is always provided
  • UNIQUE – prevents duplicate values
  • CHECK – enforces custom conditions
