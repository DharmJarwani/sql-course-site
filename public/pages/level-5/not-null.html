NOT NULL – Require Mandatory Values

The NOT NULL constraint ensures that a column cannot have NULL values.

What is NOT NULL?

NOT NULL is a column-level constraint that enforces presence — every row must contain a value for that column.

  • Ensure essential fields like username or email are always provided
  • Avoid NULL-related logic in application code
  • Improve data integrity and consistency

NOT NULL Constraint Syntax

CREATE TABLE Users (
  UserID INT PRIMARY KEY,
  Username VARCHAR(50) NOT NULL,
  Email VARCHAR(100)
);

Northwind Example – Non-null Country

Suppose we want to ensure that every Customer in the database has a Country value. 

ALTER TABLE Customers
ALTER COLUMN Country VARCHAR(50) NOT NULL;
