ALIAS – Rename Columns or Tables

Use AS to create easy-to-read names for columns or tables in query output.

When do we use ALIAS?

Use aliases when you want to rename a column or table temporarily within a query. This helps with:

  • Making query output more readable
  • Using shorthand names in joins or subqueries
  • Renaming calculated or aggregated values

Syntax – Column and Table Alias

SELECT column_name AS alias_name
FROM   table_name AS short_name;

Example 1 – Rename a Column

SELECT FirstName AS Name,
       City AS Location
FROM   Customers;

Example 2 – Rename a Table

SELECT C.CompanyName,
       O.OrderDate
FROM   Customers AS C
JOIN   Orders AS O
ON     C.CustomerID = O.CustomerID;

Example 3 – Alias for Aggregated Result

SELECT CustomerID,
       COUNT(*) AS TotalOrders
FROM   Orders
GROUP BY CustomerID;

Example 4 – Alias in Expressions

SELECT ProductName,
       UnitPrice * UnitsInStock AS StockValue
FROM   Products;

Example 5 – Join with Alias

SELECT E.FirstName,
       R.RegionDescription
FROM   Employees AS E
JOIN   Regions AS R
ON     E.RegionID = R.RegionID;
