INDEX – Speed Up Data Retrieval

An index improves the speed of data lookups in a table, similar to a book's index.

What is an Index?

An index in SQL is a performance-tuning method that allows faster retrieval of records from a table.

  • Quickly search for rows by column values (like `CustomerID`, `Email`, etc.)
  • Used heavily in large tables to avoid full table scans
  • Can be created on one or more columns

Basic Syntax to Create an Index

CREATE INDEX idx_customer
ON Customers(LastName);

Example – Index on Customers LastName

This creates an index on the LastName column of the Customers table to speed up queries that filter or sort by last name.

Index NameTableColumn
idx_customerCustomersLastName

Good to Know

  • Indexes do not affect the actual data in the table
  • Too many indexes can slow down insert/update/delete
  • Primary and unique keys automatically create indexes
