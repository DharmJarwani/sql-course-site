INNER JOIN – Connect Related Tables

Use INNER JOIN to fetch combined data from two or more tables based on a matching condition.

What is a JOIN and when to use INNER JOIN?

A JOIN is used in SQL to combine rows from two or more tables, based on a related column between them. INNER JOIN returns only the records that have matching values in both tables.

Use INNER JOIN when:

  • You need data from multiple tables (like Orders + Customers)
  • Only rows with matching values should be included
  • You want to avoid nulls from unmatched records (unlike LEFT JOIN)

INNER JOIN Syntax

SELECT table1.columnA, table2.columnB
FROM   table1
INNER JOIN table2
ON table1.common_column = table2.common_column;

Example 1 – Orders with Customer Names

SELECT o.OrderID, c.CompanyName, o.OrderDate
FROM   Orders o
INNER JOIN Customers c
ON o.CustomerID = c.CustomerID;

Example 2 – Order Details with Product Info

SELECT od.OrderID, p.ProductName, od.Quantity
FROM   [Order Details] od
INNER JOIN Products p
ON od.ProductID = p.ProductID;

Example 3 – Employee Who Took Each Order

SELECT o.OrderID, e.FirstName, e.LastName
FROM   Orders o
INNER JOIN Employees e
ON o.EmployeeID = e.EmployeeID;

Example 4 – Products and Their Categories

SELECT p.ProductName, c.CategoryName
FROM   Products p
INNER JOIN Categories c
ON p.CategoryID = c.CategoryID;

Example 5 – Supplier of Each Product

SELECT p.ProductName, s.CompanyName AS Supplier
FROM   Products p
INNER JOIN Suppliers s
ON p.SupplierID = s.SupplierID;
