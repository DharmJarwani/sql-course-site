SQL Data Types – Define What Your Data Looks Like

Each column in a table must be assigned a type: number, text, date, etc.

What Are Data Types?

Data types define the kind of values that can be stored in each column. Choosing the right data type improves performance, enforces data integrity, and reduces storage size.

Common SQL Data Types

TypeDescriptionExample
INTWhole numbers42, -10, 999
DECIMALFixed-point numbers10.99, 5.75
VARCHAR(n)Variable-length text (n = max chars)'Hello', 'John Doe'
DATECalendar dates'2024-12-31'
DATETIMEDate and time together'2024-12-31 23:59:59'
BITBoolean value (0 or 1)1 (true), 0 (false)

Best Practices

  • Use INT for numeric IDs and counts
  • Use VARCHAR with appropriate length (e.g. VARCHAR(100))
  • Prefer DATE over storing dates as strings
  • Always use DECIMAL for prices and money

Tip – Data Type Differences

CHAR is fixed-length, while VARCHAR is variable-length.
FLOAT can be imprecise for calculations — use DECIMAL if precision matters.
