Stored Procedures – Reusable SQL Logic

Bundle your SQL logic into callable, reusable routines.

What is a Stored Procedure?

A stored procedure is a precompiled SQL program saved in the database. You can execute it by name, often with input/output parameters.

  • Encapsulates complex queries or logic
  • Improves performance (compiled once)
  • Promotes reusability and consistency
  • Enhances security with controlled access

Basic Syntax

CREATE PROCEDURE ProcedureName
    @Param1 INT,
    @Param2 VARCHAR(50)
AS
BEGIN
    -- SQL logic here
END;

Example 1 – Procedure for Customer Orders

CREATE PROCEDURE GetCustomerOrders
    @CustomerID NCHAR(5)
AS
BEGIN
    SELECT OrderID, OrderDate
    FROM Orders
    WHERE CustomerID = @CustomerID;
END;

Example 2 – Execute Stored Procedure

EXEC GetCustomerOrders 'ALFKI';

Example 3 – Procedure with Total Sales

CREATE PROCEDURE TotalSalesByCountry
AS
BEGIN
    SELECT ShipCountry,
           SUM(Freight) AS TotalFreight
    FROM Orders
    GROUP BY ShipCountry;
END;
