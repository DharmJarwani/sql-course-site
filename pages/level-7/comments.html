Comments – Annotate Your SQL Code

Make your SQL easier to read and maintain by adding helpful comments.

Why Use Comments?

  • Explain logic behind complex queries
  • Temporarily disable part of the SQL during testing
  • Improve collaboration in team environments
  • Document assumptions or limitations

Types of SQL Comments

  • Single-line: Starts with --
  • Multi-line: Enclosed between /* and */

Example 1 – Single-line Comment

-- Select customer orders from the USA
SELECT * 
FROM Orders
WHERE ShipCountry = 'USA';

Example 2 – Multi-line Comment

/*
Get orders shipped to Germany.
We are only interested in the columns relevant to shipping.
*/
SELECT OrderID, ShipName, ShipCountry
FROM Orders
WHERE ShipCountry = 'Germany';

Example 3 – Disable Query Line

SELECT * 
FROM Customers;
-- WHERE Country = 'Brazil';
