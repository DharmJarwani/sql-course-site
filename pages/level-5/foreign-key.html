FOREIGN KEY – Linking Tables Together

A FOREIGN KEY connects rows in one table to rows in another — enabling relational data.

What is a FOREIGN KEY?

A FOREIGN KEY is a column (or combination of columns) that creates a link between two tables by referencing the PRIMARY KEY of another table.

  • Maintains **referential integrity** across tables
  • Prevents invalid data from being inserted into the referencing table
  • Creates a **parent-child relationship** between tables
  • Used heavily in normalized databases for efficient structure

Foreign Key Syntax

CREATE TABLE Orders (
  OrderID INT PRIMARY KEY,
  CustomerID INT,
  FOREIGN KEY (CustomerID)
    REFERENCES Customers(CustomerID)
);

Northwind Example – Customers & Orders

In the Northwind database, the Orders table references the Customers table using a foreign key. Each order is associated with a valid customer.
