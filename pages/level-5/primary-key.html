PRIMARY KEY – Uniquely Identify Each Row

The foundation of data integrity – every table needs a way to uniquely identify its records.

What is a PRIMARY KEY?

A PRIMARY KEY is a constraint that guarantees each row in a table is unique and identifiable.

  • Each table can have **only one** primary key
  • It can consist of a single column or a combination of columns
  • Primary key columns **cannot contain NULLs**
  • Ensures **uniqueness** and helps maintain **referential integrity**
  • Often used as the **target for foreign keys** in other tables
  • Automatically creates a **unique index** for faster lookup

Primary Key Syntax

CREATE TABLE Customers (
  CustomerID INT PRIMARY KEY,
  Name VARCHAR(100)
);

Northwind Example – Customers

In the Northwind database, the CustomerID is the primary key of the Customers table. This ensures that no two customers can have the same ID.
