RIGHT JOIN – Include All from Right Table

Returns all rows from the right table, with matching rows from the left.

When do we use RIGHT JOIN?

RIGHT JOIN (also called RIGHT OUTER JOIN) is used when we want to:

  • Keep all data from the right-hand table (even without a match)
  • Find "unlinked" records on the right side
  • Audit incomplete or missing relationships

Real Examples (Northwind DB):

  • List all products, even those that haven't been ordered
  • List all employees, even if they haven’t processed any orders

Typical RIGHT JOIN Pattern

SELECT columns
FROM   TableA
RIGHT JOIN TableB
ON     TableA.key = TableB.key;

Example 1 – All Products and their Orders (if any)

SELECT p.ProductName, od.OrderID
FROM   [Order Details] od
RIGHT JOIN Products p
ON     p.ProductID = od.ProductID;

How is RIGHT JOIN Different from LEFT JOIN?

LEFT JOIN includes all rows from the left table and matches from the right.
RIGHT JOIN includes all rows from the right table and matches from the left.

The direction matters based on which table you want to keep all rows from.
