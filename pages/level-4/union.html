UNION – Combine Results from Multiple Queries

Use UNION to merge rows from two or more queries into one result set.

When do we use UNION?

UNION allows combining the output of two or more SELECT statements, as long as they have the same number of columns and compatible data types.

Typical uses in Northwind DB:

  • Get customers and suppliers from different tables in one list
  • Merge results of orders from different countries
  • Combine employees and contractors into a single directory
  • Union historical and current data

UNION Syntax

SELECT column1, column2 FROM TableA
UNION
SELECT column1, column2 FROM TableB;

Example 1 – Customers and Suppliers in One List

SELECT ContactName, 'Customer' AS Type
FROM Customers
UNION
SELECT ContactName, 'Supplier'
FROM Suppliers;

Example 2 – Orders Shipped to USA or Canada

SELECT OrderID, ShipCountry
FROM Orders
WHERE ShipCountry = 'USA'
UNION
SELECT OrderID, ShipCountry
FROM Orders
WHERE ShipCountry = 'Canada';

Example 3 – Contact Titles from Two Tables

SELECT ContactTitle FROM Customers
UNION
SELECT ContactTitle FROM Suppliers;

Example 4 – All Cities from Customers and Suppliers

SELECT City FROM Customers
UNION
SELECT City FROM Suppliers;

Example 5 – Union with Different Labels

SELECT CompanyName, 'Customer' AS Type
FROM Customers
UNION
SELECT CompanyName, 'Supplier'
FROM Suppliers;

When should you use UNION?

UNION is best when you want to combine rows from multiple sources into a single set — especially when the tables are similar but not directly connected via relationships.

Unlike JOIN, it doesn't combine columns — it combines rows. Make sure each SELECT has the same number and type of columns.
